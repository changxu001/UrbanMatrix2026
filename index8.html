Welcome to Urban Matrix

Urban Matrix is a spatial analysis platform for evaluating Transit-Oriented Development potential across station areas. It combines land use, demographics, employment, walkability, transit access, and park coverage into a single composite score — helping planners compare and prioritize sites with consistent, data-driven metrics.

Our vision is to make rigorous TOD analysis accessible and repeatable — every station area profiled here was generated using the same standardized ArcGIS Pro toolbox, so results are directly comparable across cities and over time.

✕

Methodology

Urban Matrix profiles transit station areas across twelve standardized metrics, each derived from a consistent set of public and licensed data sources, then combined into composite sub-scores and an overall UM Score for comparison across sites.

Terminology

  • Station Area — a planning boundary (typically a ¼–½ mile radius or custom polygon) drawn around a transit stop or future transit investment.
  • UM Score — the sum of twelve 0–10 sub-scores, for a maximum possible 120 points.
  • Sub-score — an individual 0–10 rating for one metric (e.g. Walk Score, FAR), normalized relative to all station areas currently loaded.

Calculation Method

Each raw metric (e.g. Population_Density, Walk_Score) is normalized using min–max scaling across all station areas in the loaded dataset:

sub_score = ((value − min) / (max − min)) × 10

The Sustainable Commute metric is calculated as 1 − Drive_Alone_PCT before normalization, so areas with lower drive-alone commute shares score higher.

Assumptions

  • Min–max normalization means sub-scores are relative to the current dataset — adding or removing station areas will shift other areas' scores.
  • Average FAR is an estimated proxy based on parcel and building footprint data, not authoritative assessor records — treat as directional, not exact.
  • Transit frequency metrics rely on the most recently available GTFS/Transitland export and may not reflect very recent schedule changes.

Running Your Own Analysis

  1. Open the UrbanMatrix toolbox (UrbanMatrix_All_Profile_Tool.pyt) in ArcGIS Pro.
  2. Provide a polygon feature class of your station area boundaries as input.
  3. Run the All Profile tool — it sequentially runs each sub-tool (Land Use/FAR, Population, Employment, Parks/POI/Trails, Walk/Bike Score, Transit Coverage, Transit Frequency, Historic Places) and merges results onto your boundaries.
  4. Run UM_Score.pyt on the merged output to calculate sub-scores and the total UM Score.
  5. Export the scored layer to GeoJSON: right-click the layer → Data → Export Features → set output type to GeoJSON, or use arcpy.conversion.FeaturesToJSON with geoJSON=True.
  6. Load the exported .geojson file into Urban Matrix using the Upload GeoJSON File button, or replace data/UM_Database.geojson to make it the new default dataset.
